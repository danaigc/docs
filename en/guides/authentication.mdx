## API Keys

Egg API uses API keys for authentication. Each request must include your API key in the `Authorization` header.

## Getting Your API Key

<Steps>
  <Step title="Sign up">
    Create an account at [eggapi.ai](https://eggapi.ai)
  </Step>
  <Step title="Go to Dashboard">
    Navigate to the [API Keys section](https://eggapi.ai/dashboard)
  </Step>
  <Step title="Create a Key">
    Click "Create API Key" and give it a descriptive name
  </Step>
  <Step title="Copy and Store">
    Copy your key immediately - it won't be shown again
  </Step>
</Steps>

## Using Your API Key

Include your API key in the `Authorization` header with the `Bearer` prefix:

<CodeGroup>

```bash cURL
curl https://api.eggapi.ai/v1/generate \
  -H "Authorization: Bearer egg_sk_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"model": "nanobanana", "prompt": "..."}'
```

```python Python
import requests

headers = {
    "Authorization": "Bearer egg_sk_your_api_key_here",
    "Content-Type": "application/json"
}

response = requests.post(
    "https://api.eggapi.ai/v1/generate",
    headers=headers,
    json={"model": "nanobanana", "prompt": "..."}
)
```

```javascript JavaScript
const response = await fetch("https://api.eggapi.ai/v1/generate", {
  method: "POST",
  headers: {
    "Authorization": "Bearer egg_sk_your_api_key_here",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({ model: "nanobanana", prompt: "..." })
});
```

</CodeGroup>

## Security Best Practices

<Warning>
  Never expose your API key in client-side code, public repositories, or share it with others.
</Warning>

### Do's

- Store API keys in environment variables
- Use server-side code to make API calls
- Rotate keys periodically
- Create separate keys for different environments (dev, staging, prod)

### Don'ts

- Hardcode API keys in your source code
- Commit API keys to version control
- Share API keys in chat or email
- Use the same key for all environments

## Environment Variables

Store your API key in an environment variable:

```bash .env
EGGAPI_API_KEY=egg_sk_your_api_key_here
```

Access it in your code:

<CodeGroup>

```python Python
import os
import requests

api_key = os.environ.get("EGGAPI_API_KEY")

response = requests.post(
    "https://api.eggapi.ai/v1/generate",
    headers={"Authorization": f"Bearer {api_key}"},
    json={...}
)
```

```javascript Node.js
const apiKey = process.env.EGGAPI_API_KEY;

const response = await fetch("https://api.eggapi.ai/v1/generate", {
  headers: { "Authorization": `Bearer ${apiKey}` },
  ...
});
```

</CodeGroup>

## Authentication Errors

If authentication fails, you'll receive a `401 Unauthorized` response:

```json
{
  "data": null,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key"
  }
}
```

Common causes:
- Missing `Authorization` header
- Missing `Bearer` prefix
- Invalid or revoked API key
- Typo in the API key

## Managing API Keys

From your dashboard, you can:

- **Create** new API keys
- **View** existing keys (partial display only)
- **Revoke** compromised keys
- **Track** usage per key

<Note>
  If you suspect your API key has been compromised, revoke it immediately and create a new one.
</Note>
